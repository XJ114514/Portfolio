The “[i]” operator means move the pointer to ith positions for example (stuff[0] = *(stuff+0) which means moving the ptr by 0 times and the ptr is still pointing to the first element.)

12. What is the output if we run GDB's backtrace (abbreviated "bt") command inside the function F() in Blowfish.c the first time F() is called? Briefly explain the output of the command in your own words

(gdb) bt

#0  F (ctx=0x61d730, x=1753098189) at blowfish.c:550

#1  0x00000000004017dc in Blowfish_Encrypt (ctx=0x61d730, xl=0x61d6ec, xr=0x61d6e8) at blowfish.c:602

#2  0x0000000000401a30 in Blowfish_Init (ctx=0x61d730, key=0x405000 "LAME_KEY", keyLen=8) at blowfish.c:754

#3  0x00000000004015f0 in main () at GDBassign.c:383

Explanation:

          The #0 shows the first/current stack - the function call of F() in function Blowfish_Encrypt(). And we are running this stack.

          The #1 shows the second stack - the function call of Blowfish_Encrypt() in Blowfish_Init(). And we are going to continue running the second stack after finishing the first/current stack.

          The #2 shows the third stack – the function call of Blowfish_Init() in main(). And we are going to continue running the third stack after finishing the second stack.

          The #3 shows the fourth/last stack – the execution of main() function in GDBassign.c. And we are going to continue running the fourth stack after finishing the third stack.

          The structure of the stacks should be like:

Main(){

Blowfish_Init(){

                    Blowfish_Encrypt(){

                              F(){

}
